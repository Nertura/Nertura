# Nertura — User Personas

> Detailed personas representing every primary user type on the Nertura platform. Each persona drives feature prioritization, UX decisions, and go-to-market messaging.

---

## Persona Overview

| Persona | Primary Role | Organization Type | Nertura Tier |
|---------|--------------|-------------------|--------------|
| Farmer | Owner-operator | Individual farm | Starter / Professional |
| Farm Manager | Operations lead | Farm or estate | Professional / Business |
| Cooperative | Member organization admin | Cooperative / association | Business |
| Agricultural Company | Corporate agribusiness | Ag company / conglomerate | Business / Enterprise |
| Supplier | Input provider | Seed, fertilizer, equipment | Professional / Business |
| Exporter | Trade & logistics | Export company | Business / Enterprise |
| Administrator | Platform & IT admin | Any organization | All tiers |

## 1. Farmer

### Profile

| Attribute | Detail |
|-----------|--------|
| **Name** | Maria Santos |
| **Age** | 42 |
| **Location** | Rural region, primary crop zone |
| **Farm size** | 45 hectares |
| **Crops** | Corn, soybeans, vegetables (rotation) |
| **Tech comfort** | Moderate — uses smartphone daily, limited desktop experience |
| **Languages** | Native language + basic English |

### Background

Maria inherited and expanded her family farm. She manages day-to-day operations herself with seasonal hired help. She sells through a local cooperative and occasionally direct to regional buyers. She tracks expenses in notebooks and spreadsheets but lacks a unified system.

### Goals

- Increase yield while reducing input waste (fertilizer, water, pesticides)
- Know when to irrigate, spray, or harvest without guessing
- Track costs and revenue per field and crop season
- Get fair market prices and find reliable buyers
- Spend less time on paperwork and more time in the field

### Pain Points

- Too many disconnected apps (weather, market prices, WhatsApp groups)
- Cannot afford enterprise farm software
- Disease and pest problems detected too late
- Unpredictable weather ruins plans
- Difficulty proving quality to buyers for premium pricing

### Nertura Usage

| Module | Usage Pattern |
|--------|---------------|
| Dashboard | Daily check — weather, alerts, tasks |
| Farm Management | Field boundaries, soil records |
| Crop Management | Planting schedules, growth tracking |
| Weather Intelligence | Daily forecasts, frost/heat alerts |
| Irrigation | Schedule and monitor water usage |
| AI Assistant | Ask questions in natural language ("Should I spray today?") |
| Marketplace | List harvest; compare buyer offers |
| Mobile app | Primary interface — used in field, often offline |

### Success Metrics

- 15%+ reduction in water and input costs
- Early disease detection before crop damage
- Complete digital record for one full season
- At least one successful Marketplace transaction

### Quote

> *"I don't need more data — I need to know what to do today."*

## 7. Administrator

### Profile

| Attribute | Detail |
|-----------|--------|
| **Name** | Priya Sharma |
| **Age** | 34 |
| **Location** | Organization headquarters or IT department |
| **Role** | System administrator / IT manager |
| **Tech comfort** | Expert — IAM, SaaS admin, security |

### Background

Priya is responsible for configuring, securing, and maintaining her organization's Nertura instance. She manages user accounts, permissions, integrations, billing, and compliance settings. She may support a cooperative, agribusiness, or multi-entity agricultural company.

### Goals

- Secure, compliant platform configuration
- Efficient user onboarding and offboarding
- Role-based access aligned with org structure
- Integration with SSO, ERP, and internal systems
- Cost control and subscription management

### Pain Points

- Complex permission models across farms, regions, and roles
- Users creating accounts outside IT governance
- Audit requirements for access logs and data changes
- Integration setup requires developer resources
- Unclear billing across departments or member entities

### Nertura Usage

| Module | Usage Pattern |
|--------|---------------|
| User Management | Primary workspace — users, roles, groups |
| Billing | Subscription, invoices, usage |
| Settings | Org profile, localization, security policies |
| Integrations | API keys, webhooks, SSO (Enterprise) |
| Reports | Audit logs; usage analytics |
| Notifications | System alerts; security events |

### Role Permissions Model

| Admin Level | Capabilities |
|-------------|--------------|
| **Org Admin** | Full org settings, billing, all user management |
| **Site Admin** | Users and settings for assigned farms/sites |
| **Module Admin** | Configuration for specific modules (e.g., Marketplace) |
| **Support Admin** | User support actions; no billing or security changes |

### Success Metrics

- User provisioning time under 5 minutes
- Zero unauthorized access incidents
- SSO live for Enterprise deployment
- Complete audit trail for compliance review

### Quote

> *"The platform is powerful — my job is making sure the right people have the right access, and nothing more."*
