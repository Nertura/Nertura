# Chapter 11 — Home Page Philosophy

## Purpose

Explain **why the Nertura home page offers only three actions** — ask, upload photo, take photo — and why this is deliberate product strategy, not missing features.

---

## Principles

1. **The problem is the homepage** — not our company story
2. **Three actions cover 95% of intent** — question, existing photo, new photo
3. **ChatGPT simplicity wins** — one composer, infinite depth behind it
4. **Features are discovered through use** — not marketing grids
5. **Nothing else competes for attention**

---

## The Home Page IS the Product

On `nertura.com`, the homepage is not a **brochure**. It is the **product entry point**.

```
┌─────────────────────────────────────────┐
│              Nertura                    │
│     The AI Brain for Agriculture        │
│                                         │
│  ┌─────────────────────────────────┐  │
│  │ Ask about your crops, fields... │  │
│  │                          📷 🎤 │  │
│  └─────────────────────────────────┘  │
│                                         │
│         (nothing else competes)         │
└─────────────────────────────────────────┘
```

Supported first actions:

| Action | Farmer intent |
|--------|---------------|
| **Ask a question** | "Why is my wheat yellow?" |
| **Upload a photo** | "I already took a picture" |
| **Take a photo** | "I'm looking at the problem now" (camera capture) |

Voice and location are **secondary scaffolds** — available through intake and field context, not homepage clutter.

---

## Why NOT Feature Grids, Dashboards, or Modules

| Alternative | Why we reject it |
|-------------|------------------|
| **Feature grid homepage** | Forces farmer to choose product area before stating problem |
| **Dashboard-first** | Empty charts for guests; signals work required |
| **Pricing above fold** | Before value demonstrated — converts poorly with skeptics |
| **Blog / news carousel** | Distracts from urgency of crop problems |
| **Multi-product tabs** | Nertura is one brain, not a suite |

### ChatGPT simplicity wins because:

1. **Zero learning curve** — everyone knows how to type a question
2. **Infinite depth behind one interface** — complexity emerges when needed
3. **Mobile thumb zone** — single input fits one-handed field use
4. **SEO + word of mouth** — "just go to Nertura and ask" is shareable
5. **AI-native expectation** — 2024+ users expect composer-first AI products

Nertura follows this philosophy **because farmers are not less sophisticated than ChatGPT users — they are more skeptical and more busy.**

---

## What Happens After the First Action

The homepage is simple; the **product ladder** is deep:

```
Ask / Upload / Photo
        │
        ▼
   Short answer (guest)
        │
        ├── Continue conversation (3 free questions)
        ├── Sign up for more
        └── Add field context (logged in)
                │
                ▼
           Field OS dashboard
                │
                ▼
           Cases, memory, premium
```

**Depth is behind the composer, not beside it.**

---

## Logged-In Home Difference

Registered users with farms see the **Field OS home** (RC-2) — not the marketing composer. This is correct:

| User | Home |
|------|------|
| Guest | Google-like composer |
| Logged-in | Field-centric calm dashboard + Doctor CTA |

Both share **calm, premium, non-technical** feel. Neither shows ERP complexity first.

---

## Decision Rationale

Every element on the guest homepage must **reduce time to first question**. Analytics heatmaps consistently show: **more links above fold = lower activation**.

Agriculture marketing often shows **green fields stock photos**. Farmers scroll past them — they have their own fields. Show the **input**.

---

## Examples

### Good homepage elements

- Composer with agricultural placeholder text
- Subtle trust line: "Free · No account needed for first questions"
- Legal/footer links only below fold

### Bad homepage elements

- "Explore our 12 modules" button row
- Auto-playing drone video
- Pop-up discount before first question

---

## Best Practices

- A/B test **placeholder copy**, not layout complexity
- Measure **A1 activation** (first message) as primary homepage KPI
- Localize composer placeholder (TR/EN)

## Bad Practices

- Adding homepage sections for internal stakeholder requests
- Duplicating dashboard nav on marketing site
- Hiding composer below fold on mobile

---

## Future Considerations

- **Regional homepage variants** — crop examples in placeholder by geo
- **Seasonal prompts** — "Wheat rust season — ask about orange spots"
- **PWA install** — still composer-first after install
